docker ps → Lista containers em execução.
docker ps -a → Lista todos os containers (inclusive parados).
docker image ls → Lista todas as imagens locais.
docker build -t api . → Cria a imagem api a partir do Dockerfile no diretório atual.
docker build -t api:v1 . → Cria a imagem api com tag de versão v1.
docker run -p 3333:3333 api → Executa a imagem api, mapeando porta 3333 do host para 3333 do container.
docker run -p 3333:3333 -d api → Executa a imagem api em background (modo detached).
docker run -p 3333:3333 -d api:v2 → Executa em background a imagem api com a tag v2.
docker logs [ID/NOME] → Mostra os logs de um container.
docker history [ID/NOME da imagem] → Mostra o histórico de camadas de uma imagem.
docker pause [ID/NOME] → Pausa a execução de um container.
docker unpause [ID/NOME] → Retoma a execução de um container pausado.
docker stop [ID/NOME] → Para um container em execução.
docker start [ID/NOME] → Inicia um container parado.
docker rm [ID/NOME] → Remove um container parado.
docker rm -f [ID/NOME] → Força a remoção de um container em execução.
docker rmi [ID/NOME da imagem] → Remove uma imagem local.
docker exec -it [ID/NOME] /bin/sh → Abre um shell dentro do container (quando há /bin/sh).
docker exec -it [ID/NOME] bash → Abre um shell dentro do container (quando há bash).
